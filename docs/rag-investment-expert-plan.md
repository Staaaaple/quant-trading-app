# RAG 投资顾问专家知识库搭建方案

> 目标：训练 RAG 成为具备「选股能力 + 资产组合能力 + 买卖节点判断能力」的投资顾问专家

---

## 一、核心能力拆解

### 1.1 三大核心能力

```
投资顾问专家
├── 🎯 选股能力 (Stock Picking)
│   ├── 基本面分析：财报解读、行业地位、护城河
│   ├── 技术面分析：趋势、支撑阻力、量价关系
│   ├── 估值分析：PE/PB/PEG/DCF估值
│   └── 因子选股：质量、动量、价值、成长因子
│
├── ⚖️ 资产组合能力 (Portfolio Construction)
│   ├── 战略配置：股债商现比例（SAA）
│   ├── 战术配置：行业轮动、风格切换（TAA）
│   ├── 风险分散：相关性分析、集中度控制
│   └── 再平衡：偏离度触发、定期调整
│
└── ⏰ 买卖节点判断 (Timing)
    ├── 买入信号：估值低位、趋势突破、催化剂出现
    ├── 卖出信号：估值高位、趋势破位、目标达成
    ├── 止损纪律：固定比例、波动率止损、时间止损
    └── 仓位管理：金字塔加仓、倒金字塔减仓
```

### 1.2 知识库与能力的映射

| 能力 | 主要知识源 | 辅助知识源 |
|------|-----------|-----------|
| **选股** | 个股分析案例库、估值方法论 | 论文片段（因子投资） |
| **组合** | 资产配置规则库、组合案例库 | 市场规律（美林时钟） |
| **买卖节点** | 择时案例库、技术信号库 | 历史案例（成功/失败交易） |

**文档结构**：
```
[rule_id]: alloc_saa_conservative_recovery
[类型]: SAA-战略配置
[适用画像]: 保守型投资者
[适用周期]: 复苏期

[配置方案]:
├── 股票: 30%（沪深300ETF 20% + 中证500ETF 10%）
├── 债券: 50%（国债30% + 高等级信用债20%）

└── 现金: 10%（货币基金）

[配置理由]:
├── 保守型股票上限30%，控制权益风险
├── 复苏期股票性价比最高，但保守型不追高
├── 债券50%提供稳定收益和下行保护
├── 黄金10%对冲地缘政治不确定性
└── 现金10%保持流动性，等待更好买点

[调整触发]:
├── 周期切换: 复苏→过热，股票降至25%
├── 偏离度: 股票占比超35%或低于25%时再平衡
└── 寿命到期: 组合寿命18个月，到期重新评估
```

### 2.5 知识源五：行为金融学案例库 (behavioral_cases)

**定位**：投资者常见心理偏差及应对策略

**数据来源**：
- Kahneman/Tversky 经典实验
- 中国市场散户行为研究
- 投资大师的经验教训

**文档结构**：
```
[case_id]: behavior_loss_aversion_001
[偏差]: 损失厌恶 (Loss Aversion)
[来源]: Kahneman & Tversky, 1979

[现象描述]:
"投资者持有亏损股票不愿卖出（套牢），却过早卖出盈利股票（止盈）。
 实验数据显示：人们对同等金额损失的痛感是盈利快感的2.5倍。"

[中国市场案例]:
├── 2015年股灾：大量散户在下跌50%后仍不愿止损
├── 2021年核心资产：茅台从2600跌到1500，散户一路补仓
└── 数据：A股散户持仓亏损股票平均持有时间比盈利股票长40%

[应对策略]:
├── 预设止损线：买入前确定止损位，触发后机械执行
├── 仓位控制：单只股票不超过20%，避免情感绑定
├── 定期审视：每季度评估持仓，不问成本只看未来
└── 反向操作：设置"止盈不止损"规则，让利润奔跑

[系统对策]:
├── 画像识别：损失厌恶高分用户→自动收紧止损线
├── 预警推送：持仓亏损超8%时推送"止损提醒"
└── 替代策略：推荐低波动策略，降低触发损失厌恶的频率
```

### 2.6 知识源六：基础金融常识库 (finance_basics)

**定位**：投资小白的"词典"，解释基础概念

**数据来源**：
- 证监会/交易所投资者教育
- 经典入门书籍
- 自行编写的标准化解释

**文档结构**：
```
[entry_id]: basic_pe_ratio
[主题]: 市盈率 (PE)
[难度]: 入门
[类别]: 估值指标

[定义]:
"市盈率 = 股价 / 每股收益(EPS)
 表示投资者愿意为每1元盈利支付多少价格"

[通俗解释]:
"PE就像'回本年限'。PE=20意味着按当前盈利，20年回本。
 但注意：盈利会增长，实际回本可能更短。"

[使用方法]:
├── 横向比较：同行业内PE低的可能被低估
├── 纵向比较：与自身历史PE比，低于均值可能便宜
├── 绝对判断：PE<15通常便宜，PE>30通常贵（非绝对）
└── 结合增长：PEG=PE/盈利增速，PEG<1更合理

[注意事项]:
├── ❌ 亏损企业PE为负，无法使用
├── ❌ 周期性行业PE低时可能见顶（盈利高点）
├── ❌ 不同行业PE不可比（银行PE 5x vs 科技PE 50x）
└── ✅ 结合PB、ROE一起看更准确

[实战案例]:
"2024年1月，贵州茅台PE 28x，处于历史30%分位，
 结合其15%的盈利增速，PEG=1.87，估值合理偏低。"

[相关概念]: ["PB", "PEG", "估值", "每股收益"]
```

## 四、数据模型

```python
# models/rag_investment_knowledge.py

class StockAnalysisCase(Base):
    """个股分析案例"""
    __tablename__ = "stock_analysis_cases"
    
    id = Column(Integer, primary_key=True)
    case_id = Column(String(64), unique=True, index=True)
    
    # 标的
    symbol = Column(String(16), index=True)
    name = Column(String(64))
    analysis_date = Column(Date)
    
    # 分析内容
    investment_logic = Column(Text)      # 投资逻辑
    key_metrics = Column(JSON)           # 关键指标
    valuation_analysis = Column(Text)    # 估值分析
    catalysts = Column(JSON)             # 催化剂
    
    # 结论
    recommendation = Column(String(16))  # 买入/持有/卖出
    target_price = Column(Float)
    stop_loss_price = Column(Float)
    suggested_position = Column(String(32))
    
    # 内容（用于Embedding）
    content = Column(Text)


class TimingSignalCase(Base):
    """择时信号案例"""
    __tablename__ = "timing_signal_cases"
    
    id = Column(Integer, primary_key=True)
    signal_id = Column(String(64), unique=True, index=True)
    
    # 信号信息
    signal_type = Column(String(32))     # 均线金叉/突破/估值低位等
    symbol = Column(String(16))
    signal_date = Column(Date)
    
    # 市场环境
    market_cycle = Column(String(16))
    market_conditions = Column(Text)
    
    # 信号描述
    signal_description = Column(Text)
    entry_plan = Column(JSON)            # 建仓/加仓/止损/目标
    
    # 结果
    actual_outcome = Column(Text)        # 实际走势
    lessons_learned = Column(Text)       # 经验教训
    
    # 内容（用于Embedding）
    content = Column(Text)


class AllocationRule(Base):
    """资产配置规则"""
    __tablename__ = "allocation_rules"
    
    id = Column(Integer, primary_key=True)
    rule_id = Column(String(64), unique=True, index=True)
    
    rule_type = Column(String(16))       # SAA/TAA
    risk_profile = Column(String(16))    # 保守/稳健/积极/激进
    market_cycle = Column(String(16))    # 复苏/过热/滞胀/衰退
    
    allocation = Column(JSON)            # 资产比例
    rationale = Column(Text)             # 配置理由
    backtest_summary = Column(JSON)      # 历史回测
    rebalance_triggers = Column(JSON)    # 调整触发条件
```

## 五、实施优先级

### Phase 1: 核心能力（2周）

**目标**：让RAG能回答"买什么"和"怎么配"

1. **个股分析案例库**（20个典型案例）
   - 茅台、宁德时代、比亚迪、中芯国际等
   - 每个案例包含：基本面+估值+催化剂+操作建议

2. **资产配置规则库**（16种组合）
   - 4种画像 × 4种周期 = 16个标准组合
   - 每个组合包含：比例+理由+回测+调整规则

3. **基础常识库**（50个核心概念）
   - PE/PB/ROE/夏普比率/最大回撤等
   - ETF/基金/股票/债券等工具解释

### Phase 2: 择时能力（2周）

**目标**：让RAG能判断"什么时候买/卖"

1. **择时信号案例库**（30个历史信号）
   - 均线金叉死叉、突破回踩、估值低位等
   - 每个信号包含：环境描述+操作计划+实际结果

2. **行为金融案例库**（20个偏差案例）
   - 损失厌恶、从众、过度自信等
   - 每个偏差包含：现象+案例+应对策略

### Phase 3: 深度理论（2周）

**目标**：让RAG回答有学术深度

1. **论文片段库**（20篇核心论文）
   - 下载PDF→解析→分块→Embedding
   - 重点：因子投资、资产配置、行为金融

2. **组合诊断能力**
   - 用户上传持仓→RAG诊断→给出调仓建议
