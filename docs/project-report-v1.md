# QuantEvo 智投助手 — 项目全链路报告

> **版本**: V1.0  
> **日期**: 2026-06-04  
> **定位**: 面向投资小白的 AI 智能资产配置与策略推荐系统  
> **核心理念**: "像体检一样做投资" — 先画像、再诊断市场、最后匹配策略

---

## 一、产品定位与目标用户

### 1.1 为什么做这个产品

中国散户投资者面临的核心痛点：

| 痛点 | 具体表现 |
|------|---------|
| **不知道买什么** | 面对 5000+ 股票、1000+ 基金，无从选择 |
| **不知道什么时候买/卖** | 追涨杀跌，高买低卖 |
| **不知道自己的风险承受力** | 明明保守却买了高波动股票，跌一点就恐慌割肉 |
| **策略没有生命周期意识** | 一个策略用到底，不知道何时失效 |
| **信息过载** | 财经新闻、研报、大 V 观点太多，无法判断 |

**QuantEvo 的解决思路**：把专业量化机构的"投研流程"产品化，让投资小白也能获得"适合自己的、有生命力的、动态调整的投资组合"。

### 1.2 目标用户

- **主力用户**: 25-40 岁、有一定积蓄（5-50万）、想理财但不懂投资的上班族
- **特征**: 买过余额宝/基金、被股票伤过、想要"省心"的理财方式
- **核心诉求**: "告诉我买什么、买多少、什么时候换"

### 1.3 与同类产品的差异

| 维度 | 雪球/同花顺 | 智能投顾（支付宝/理财通） | **QuantEvo** |
|------|-----------|------------------------|-------------|
| 画像深度 | 无 | 3-5题简单问卷 | **15题心理+行为画像** |
| 市场判断 | 用户自己看 | 黑盒 | **五层市场信号仪表盘** |
| 策略来源 | 大 V 推荐 | 固定基金组合 | **AI 发现+论文支撑+寿命管理** |
| 动态调整 | 无 | 定期再平衡 | **策略寿命预警+自动替换** |
| 教育属性 | 碎片化 | 无 | **每个操作附带原理解释** |

### 2.2 核心链路详解

#### 链路一：用户画像（"认识自己"）

**目的**: 量化用户的风险偏好、行为特征、资金属性，生成结构化画像向量。

**流程**:

```
首页点击"开始画像"
    │
    ▼
Step 1: 基本信息（3题）
    ├── 资金规模：1万以下 / 1-10万 / 10-50万 / 50万以上
    ├── 投资经验：没买过 / 买过基金 / 买过股票 / 都买过
    └── 收入稳定性：不稳定 / 一般 / 稳定
    │
    ▼
Step 2-8: 情景心理题（7题）—— 核心差异化
    ├── 损失厌恶："跌到8000怎么办" → 割肉/持有/加仓
    ├── 从众倾向："朋友推荐跟不跟" → 不跟/小跟/重仓跟
    ├── 过度自信："连涨3个月归因" → 运气/能力/市场好
    ├── 延迟满足："2万奖金怎么用" → 花掉/存定期/投资
    ├── 锚定效应："解套心理" → 解套就卖/看趋势/不管成本
    ├── 资金安全感："急用钱怎么办" → 借钱/提前取/有备用金
    └── 时间感知："多久看不到收益能接受" → 1月/3月/半年/1年
    │
    ▼
画像结果页
    ├── 标签："积极型长期投资者" / "保守型稳健投资者" 等
    ├── 六维雷达图：风险承受 / 损失厌恶 / 从众 / 过度自信 / 延迟满足 / 安全感
    ├── 建议配置预览（股票/债券/商品/现金比例）
    └── [查看完整组合] ──→ /portfolio
```

**输出**: `profile_vector`（15维结构化向量）

```python
{
  risk_tolerance,      # 风险承受力
  loss_aversion,       # 损失厌恶
  herding_tendency,    # 从众倾向
  overconfidence,      # 过度自信
  delayed_gratification, # 延迟满足
  security_need,       # 资金安全感
  time_horizon_score,  # 投资期限
  experience_level,    # 经验水平
  capital_tier,        # 资金规模
  income_stability,    # 收入稳定性
  debt_pressure,       # 负债压力
  information_processing, # 信息处理能力
  social_pressure,     # 社交压力
  emergency_response,  # 应急反应
  anchoring_effect,    # 锚定效应
  diversification_preference, # 分散偏好
  stop_loss_discipline, # 止损纪律
  emotional_stability, # 情绪稳定性
}
```

**为什么做 15 题而不是 3 题**：
- 3 题只能区分"保守/稳健/积极"，无法捕捉行为偏差
- 15 题可以识别"明明保守却爱追高"的矛盾型用户
- 行为金融学研究表明：行为偏差比风险偏好更能预测投资结果

**经济周期可视化**:

点击"查看经济周期"展开四象限图：
- 横轴：经济增长（左低右高）
- 纵轴：通胀水平（下低上高）
- 四个象限：复苏（右上绿）→ 过热（右下红）→ 滞胀（左下橙）→ 衰退（左上灰）
- 当前点位：脉冲动画圆点标注
- 用途：告诉用户"现在处于什么季节，该穿什么衣服"

**为什么需要五层而不是只看大盘**：
- 散户只看上证指数，但指数涨跌不告诉你"为什么涨/为什么跌"
- 五层信号帮助用户理解：是经济好了（宏观）还是打仗了（地缘）还是行业风口（行业）
- 不同原因对应不同策略：经济好→加仓股票；打仗了→买黄金；行业风口→追主题

## 三、核心系统详解

### 3.1 用户画像系统

**功能**: 将用户的"人"转化为可计算的"向量"

**设计逻辑**:
- 不直接问"你能承受多少亏损"（用户会说谎）
- 用情景题捕捉真实行为："跌到8000怎么办"比"你能承受20%亏损吗"更准确
- 15维向量覆盖：风险偏好 + 行为偏差 + 资金属性

**画像结果应用**:
- 风险承受力 → SAA 股票上限
- 损失厌恶 → 止损线设置
- 从众倾向 → 是否需要"反从众"策略
- 时间感知 → 策略寿命偏好

---

### 3.2 五层市场信号系统

**功能**: 多维度评估市场环境，生成结构化市场状态向量

**数据来源**:
- **宏观**: akshare 采集 GDP/CPI/PPI/PMI/M2/利率/社融/房地产投资
- **地缘**: 多源新闻爬虫（财联社/新浪财经/华尔街见闻）+ NLP 情绪分析
- **行业**: 新闻提及频率 + 情绪关键词匹配
- **社会**: 八大主题（AI/老龄化/新能源/消费降级/国产替代/数字经济/绿色转型/全球化重构）
- **资产内部**: 股债利差/市场情绪/风格轮动/成交量

**经济周期判断（多模型融合）**:

```
三大模型独立判断:
├─ 美林时钟修正版: GDP + CPI + PPI → 复苏/过热/滞胀/衰退
├─ 货币信用周期: M2 + 社融 + 利率 → 宽货币宽信用/宽货币紧信用/...
└─ 库存周期: PMI + 工业企业利润 + PPI → 主动补库/被动补库/主动去库/...

融合引擎:
├─ 加权平均三个模型的坐标输出
├─ 置信度 = 模型一致性 × 数据完整度
└─ 输出: 最终周期阶段 + 置信度 + 各模型独立结果
```

**为什么需要多模型融合**：
- 单一模型在特定环境下可能失效
- 中国经济的特殊性：房地产周期、政策干预、结构性转型
- 融合可以降低误判概率，输出置信度帮助用户判断可信度

---

### 3.3 Hybrid 组合设计系统

**功能**: 将"人"和"市场"转化为"组合"

**6步编排**:

```
Step 1: SAA（战略资产配置）
    输入: 用户风险承受力 + 市场周期 + 宏观评分 + 地缘风险
    输出: 股票/债券/商品/现金比例
    逻辑: 积极+复苏+低风险 → 股票60%；保守+衰退+高风险 → 股票30%

Step 2: TAA（战术资产配置）
    输入: 行业景气度 + 社会主题 + 资产内部信号
    输出: 股票内部的行业/主题权重调整
    逻辑: 科技景气高 → 科技ETF占股票50%；AI主题强 → 超配算力标的

Step 3: 策略-标的绑定（贝叶斯优化）
    输入: 策略池（35个模板）+ 标的列表 + 用户约束
    目标: max(策略夏普 × 健康度 × 寿命 × 标的相关性)
    约束: 风险等级≤用户承受 / 适合周期∩当前周期 / 寿命≥3月 / 同质化<0.6

Step 4: 风控配置自动生成
    输入: 用户画像（特别是损失厌恶）
    输出: 止损线 / 单标的上限 / 最大回撤硬止损 / 再平衡触发阈值
    逻辑: 损失厌恶高 → 止损线8%、仓位上限20%；损失厌恶低 → 止损线15%、仓位上限30%

Step 5: 可靠性评估
    - 回测: 近3年历史数据验证
    - 压力测试: 2022熊市、2020疫情等极端场景
    - 蒙特卡洛模拟: 1000次随机路径，计算收益分布

Step 6: 组合寿命计算
    组合寿命 = min(各组件寿命)
    组合健康度 = 加权平均(各组件健康度)
```

### 3.5 策略寿命系统（部分实现）

**状态**: 核心算法已实现，前端展示已搭建，自动化监控待完善

**目的**: 解决"策略什么时候失效"的核心问题

**生物学隐喻**:

```
策略 = 生物个体
├─ DNA: 策略的底层逻辑（动量/均值回归/趋势跟踪）
├─ 代谢率: 策略对数据新鲜度的敏感程度（高频=高代谢=短寿命）
├─ 生态位: 策略适应的市场环境（牛市/熊市/震荡）
├─ 寿命: 策略边缘的有效期（月）
└─ 健康度: 策略当前运行状态（0-100）
```

**寿命计算公式**:

```python
def calculate_lifespan(metabolic_rate, niche_width, metabolic_syndrome, homogeneity_risk):
    # 基础寿命: 代谢率越高 = 需要越新鲜的数据 = 寿命越短
    base_lifespan = max(6, 60 - metabolic_rate * 120)
    
    # 生态位宽度加成: 适应场景越多 = 寿命越长
    niche_bonus = niche_width * 24
    
    # 同质化惩罚: 太多人用 = 拥挤交易 = 寿命缩短
    homogeneity_penalty = homogeneity_risk * 20
    
    raw_lifespan = base_lifespan + niche_bonus - homogeneity_penalty
    
    # 代谢综合征: 结构性脆弱 = 严重惩罚
    if metabolic_syndrome:
        raw_lifespan *= 0.6
    
    lifespan_months = max(1, round(raw_lifespan))
    
    # 寿命阶段
    if lifespan_months > 36:   phase = "年轻"
    elif lifespan_months >= 12: phase = "成熟"
    elif lifespan_months >= 3:  phase = "衰老"
    else:                       phase = "濒危"
    
    return {
        lifespan_months,
        phase,
        aging_velocity,  # 每月边缘衰减百分比
        recommendations, # 延寿建议
    }
```

**寿命影响因素**:

| 因素 | 说明 | 对寿命的影响 |
|------|------|------------|
| **代谢率** | 策略对数据新鲜度的敏感度 | 高代谢（高频）→ 短寿命 |
| **生态位宽度** | 策略适应的市场环境数量 | 宽生态位 → 长寿命 |
| **同质化风险** | 市场上相似策略的数量 | 高同质化 → 短寿命 |
| **代谢综合征** | 策略是否存在结构性缺陷 | 有 → 寿命 × 0.6 |
| **波动率** | 市场波动程度 | 高波动 → 加速衰老 |
| **周期适配** | 策略与当前周期的匹配度 | 不匹配 → 加速衰老 |

**动态监控流程（规划中）**:

```
每月1日凌晨3:00 定时任务
    │
    ├──→ 采集市场环境数据
    │   ├── 波动率20日ATR / 历史波动率百分位
    │   ├── 当前周期阶段
    │   └── 家族拥挤度（近3月超额收益）
    │
    ├──→ 遍历所有活跃策略
    │   ├── 基础寿命（静态计算）
    │   ├── × 波动率加速因子（高波1.3 / 低波0.85）
    │   ├── × 周期适配因子（匹配1.0 / 相邻0.9 / 不适0.7）
    │   └── × 拥挤度因子（sigmoid映射超额收益）
    │
    ├──→ 计算组合层面寿命
    │   ├── 组合寿命 = min(各组件寿命)
    │   └── 组合健康度 = 加权平均
    │
    ├──→ 寿命变化检测
    │   ├── 减少>20% → 黄色预警
```

## 七、设计决策记录

### 7.1 为什么从"专业量化工具"转向"投资小白工具"

**原始定位**: 面向专业量化交易者的策略开发平台  
**转向原因**: 
- 专业用户太小众，且已有成熟工具（聚宽/掘金/米筐）
- 投资小白市场更大，痛点更明确
- 产品差异化：不是"让你自己写策略"，而是"AI 帮你配好策略"

### 7.2 为什么用"寿命"概念

**灵感来源**: 生物进化论 + 量化策略的"失效"现象  
**核心洞察**: 
- 任何策略都有有效期（市场结构变化、同质化、数据分布漂移）
- 散户不知道"什么时候该换策略"，往往策略失效了还在用
- 用"寿命"概念直观传达：策略不是永恒的，需要定期体检和更换

### 7.3 为什么做 15 题画像而不是 3 题

**行为金融学依据**: 
- Kahneman & Tversky 的前景理论：人们对损失的敏感度是收益的 2.5 倍
- 直接问"你能承受 20% 亏损吗"，用户会高估自己
- 情景题"跌到 8000 怎么办"更能捕捉真实行为

### 7.4 为什么用黑白灰设计

**设计哲学**: 
- 投资是严肃的，不需要花里胡哨
- 黑白灰 = 专业、冷静、克制
- 信息层级清晰：黑色 = 重要数据，灰色 = 辅助信息
