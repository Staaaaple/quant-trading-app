# 组合寿命系统设计

**版本**: V1  
**日期**: 2026-06-03  
**核心**: 组合寿命 = f(配置有效性, 选股新鲜度, 策略有效性)，用户可见三维度系数和权重

---

## 一、核心理念

### 从单策略寿命到组合系统寿命

**旧设计**:  
- 每个策略有独立寿命  
- 组合寿命 = min(各组件寿命)  
- 组合健康度 = 加权平均(各组件健康度)

**新设计**:  
- 组合是一个系统，寿命由三个维度共同决定  
- 每个维度有独立评分、权重、寿命计算  
- 短板效应：组合寿命由最短的维度决定  
- 用户可见三个维度的详细分解

### 设计原则

1. **透明性**: 用户能看到三个维度的分数、权重、计算过程
2. **可解释性**: 每个维度衰减都有明确原因
3. **行动导向**: 寿命低时告诉用户具体该关注什么
4. **动态性**: 每月自动更新，反映最新市场状态

---

## 二、三维寿命模型

### 2.1 维度一：配置有效性寿命

**定义**: 当前资产配置方案（股债商现比例+行业分布）在当前及预期市场环境下的有效期限

**评分因素**:

| 因素 | 权重 | 计算方法 |
|------|------|---------|
| 周期匹配度 | 40% | 当前周期∈配置适应周期? 是→100分 / 相邻→60分 / 不适→20分 |
| 周期切换概率 | 30% | 预测模型输出的切换概率，概率越高分数越低 |
| 行业景气匹配 | 20% | 重仓行业景气度评分，景气度高→分数高 |
| 地缘风险冲击 | 10% | 地缘风险>70%时扣分 |

**寿命计算**:
```
配置寿命月数 = 基础值12 × (周期匹配度得分/100) × (1-切换概率)

例: 周期匹配80分 + 切换概率30% + 行业景气75分 + 地缘风险50分
    得分 = 80×0.4 + (100-30)×0.3 + 75×0.2 + 50×0.1 = 32+21+15+5 = 73分
    寿命 = 12 × 0.73 × 0.7 = 6.1月
```

**衰减信号**:
- 市场周期指标连续2周指向新周期 → 寿命加速衰减
- 地缘风险突然升高 → 立即扣分
- 重仓行业政策利空 → 该行业权重部分寿命归零

---

### 2.2 维度二：选股新鲜度寿命

**定义**: 当前持仓股票的基本面质量相对于选股时的变化程度，反映"选股结果是否还新鲜"

**评分因素**:

| 因素 | 权重 | 计算方法 |
|------|------|---------|
| 时间衰减 | 25% | 距离上次选股天数 / 90天，超过90天开始线性衰减 |
| 基本面漂移 | 35% | ROE/毛利率/净利率的变化幅度，恶化程度越大分数越低 |
| 估值变化 | 25% | PE/PB的变化，估值过高→分数低 |
| 流动性变化 | 15% | 日均成交额变化，流动性下降→分数低 |

**寿命计算**:
```
选股寿命月数 = 基础值12 × (1 - 时间衰减率) × (1 - 基本面恶化率)

例: 选股后30天 + ROE平均下降5% + PE上升20% + 成交额稳定
    时间衰减 = 30/90 = 0.33 → 得分 100×(1-0.33) = 67分
    基本面恶化 = 5%/10%(阈值) = 0.5 → 得分 100×(1-0.5) = 50分
    估值变化 = 20%/30%(阈值) = 0.67 → 得分 100×(1-0.67) = 33分
    流动性 = 稳定 → 100分
    综合得分 = 67×0.25 + 50×0.35 + 33×0.25 + 100×0.15 = 16.75+17.5+8.25+15 = 57.5分
    寿命 = 12 × 0.575 = 6.9月
```

**衰减信号**:
- 财报季更新后，某股票ROE下滑>20% → 该股票寿命单独计算
- 突发利空（产品召回、监管处罚）→ 该股票寿命归零
- 行业整体景气度下滑 → 该行业持仓寿命同步衰减

---

### 2.3 维度三：策略有效性寿命

**定义**: 当前绑定的投资策略在近期市场环境中的信号准确率表现

**评分因素**:

| 因素 | 权重 | 计算方法 |
|------|------|---------|
| 近期胜率 | 40% | 近30日信号胜率，胜率>60%→100分 / 40-60%→60分 / <40%→20分 |
| 市场状态匹配 | 30% | 当前市场状态与策略适应状态的匹配度 |
| 信号频率 | 15% | 信号过于稀疏（月均<2次）或过于频繁（月均>10次）都扣分 |
| 回撤控制 | 15% | 策略近期最大回撤，回撤<5%→100分 / 5-10%→60分 / >10%→20分 |

**寿命计算**:
```
策略寿命月数 = 基础值12 × (近期胜率得分/100) × (市场匹配得分/100)

例: 近30日胜率45% + 市场匹配70分 + 月均信号5次 + 回撤8%
    胜率得分 = 60分 (40-60%区间)
    市场匹配 = 70分
    信号频率 = 80分 (适中)
    回撤控制 = 60分 (5-10%区间)
    综合得分 = 60×0.4 + 70×0.3 + 80×0.15 + 60×0.15 = 24+21+12+9 = 66分
    寿命 = 12 × 0.66 = 7.9月
```

**衰减信号**:
- 连续5个信号错误 → 寿命加速衰减
- 市场进入策略不适应状态（如趋势策略进入震荡市）→ 寿命减半
- 策略参数失效（如均线参数不再适用）→ 触发重新优化

## 三、组合寿命聚合

### 3.1 核心公式

```
组合寿命 = min(配置寿命 × 权重配置, 选股寿命 × 权重选股, 策略寿命 × 权重策略)

默认权重:
- 配置有效性: 40%
- 选股新鲜度: 30%
- 策略有效性: 30%

权重根据用户画像动态调整:
- 保守型用户: 选股新鲜度权重提升至35%（更关心基本面）
- 积极型用户: 策略有效性权重提升至35%（更关心信号准确）
- 长期投资者: 配置有效性权重提升至45%（更关心大方向）
```

### 3.2 计算示例

```
用户画像: 稳健型，中期投资者

维度得分:
- 配置有效性: 73分 → 寿命6.1月
- 选股新鲜度: 57.5分 → 寿命6.9月
- 策略有效性: 66分 → 寿命7.9月

权重(稳健型默认):
- 配置: 40%
- 选股: 30%
- 策略: 30%

加权计算:
- 配置加权 = 6.1 × 0.4 = 2.44
- 选股加权 = 6.9 × 0.3 = 2.07
- 策略加权 = 7.9 × 0.3 = 2.37

组合寿命 = min(2.44, 2.07, 2.37) = 2.07月

向下取整: 2月

短板识别: 选股新鲜度是短板(2.07最小)
```

## 五、动态更新机制

### 5.1 更新频率

| 维度 | 更新频率 | 触发条件 |
|------|---------|---------|
| 配置有效性 | 每周 | 市场周期指标变化 |
| 选股新鲜度 | 每日 | 股价变化导致估值漂移 / 财报更新 |
| 策略有效性 | 每日 | 新信号产生后评估 |
| 组合寿命聚合 | 每日 | 任一维度更新后重新计算 |

### 5.2 加速因子

在基础寿命计算后，应用动态加速因子:

**波动率加速因子**:
- 当前波动率百分位 >150% → 寿命 × 1.3
- 当前波动率百分位 >120% → 寿命 × 1.15
- 当前波动率百分位 <50% → 寿命 × 0.85

**周期适配因子**:
- 当前周期 ∈ 策略适应周期 → 寿命 × 1.0
- 相邻周期 → 寿命 × 0.9
- 不适配周期 → 寿命 × 0.7

**拥挤度因子(sigmoid)**:
- 家族超额收益 >10% → 寿命 × 0.75（表现好→资金涌入→未来拥挤）
- 家族超额收益 >5% → 寿命 × 0.9
- 家族超额收益 <-5% → 寿命 × 1.1（表现差→资金流出→不拥挤）

### 5.3 更新示例

```
月初计算:
- 配置寿命: 6.1月
- 选股寿命: 6.9月
- 策略寿命: 7.9月
- 组合寿命: 2月

月中市场突变(波动率飙升):
- 波动率加速因子: 1.3
- 配置寿命更新: 6.1 / 1.3 = 4.7月
- 选股寿命更新: 6.9 / 1.3 = 5.3月
- 策略寿命更新: 7.9 / 1.3 = 6.1月
- 组合寿命更新: min(4.7×0.4, 5.3×0.3, 6.1×0.3) = min(1.88, 1.59, 1.83) = 1.59月

取整: 1月

预警升级: 从"2个月后调仓"变为"1个月后调仓"
```

## 六、预警与行动

### 6.1 三级预警

| 级别 | 条件 | 用户提示 | 系统动作 |
|------|------|---------|---------|
| 绿色 | 寿命≥6月 | "组合健康，继续持有" | 每日监控 |
| 黄色 | 3月≤寿命<6月 | "组合良好，关注X因素" | 推送关注提醒 |
| 红色 | 寿命<3月 | "建议准备调仓" | 生成替代方案，推送预览 |

### 6.2 调仓建议生成

当寿命<3月时，系统生成调仓建议:

```
寿命到期原因分析:
├─ 如果是配置有效性短板
│   └─ 建议: 调整股债比例 / 切换行业配置
│   └─ 生成新SAA/TAA方案
│
├─ 如果是选股新鲜度短板
│   └─ 建议: 替换基本面恶化的股票
│   └─ 生成新的选股池，保留健康持仓
│
└─ 如果是策略有效性短板
    └─ 建议: 更换策略类型或调整参数
    └─ 重新绑定策略，回测验证
```

## 七、与原设计的继承关系

| 原设计 | 新设计 | 说明 |
|--------|--------|------|
| 单策略寿命 | 策略有效性维度 | 继承并融入三维模型 |
| 组合寿命=min(各组件) | 组合寿命=min(三维加权) | 从组件维度升级为系统维度 |
| 组合健康度=加权平均 | 各维度独立评分 | 更细粒度，用户可见 |
| 动态寿命=基础×三因子 | 各维度独立计算+聚合 | 从单策略扩展到组合系统 |
| 每月1日更新 | 每日更新 | 频率提高，更及时 |
| 预警阈值固定 | 根据画像动态 | 保守型更早预警 |

## 八、实施要点

### 8.1 数据需求

| 维度 | 需要数据 | 来源 |
|------|---------|------|
| 配置有效性 | 周期判断、切换概率、行业景气 | 市场信号模块 |
| 选股新鲜度 | 财报数据、估值数据、成交数据 | akshare + 数据库 |
| 策略有效性 | 信号记录、收益记录、回撤记录 | 策略执行日志 |

### 8.2 计算性能

- 每日更新：三个维度独立计算，可并行
- 选股新鲜度最耗：需遍历所有持仓股票的基本面变化
- 优化：基本面数据缓存，仅财报季全量更新

### 8.3 前端展示

- 主卡片：寿命月数 + 进度条 + 短板原因一句话
- 详情页：三维度展开 + 分数 + 权重 + 计算过程 + 具体建议
- 调仓页：寿命到期时展示替代方案对比
