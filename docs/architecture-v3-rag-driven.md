# QuantEvo V3 架构设计 — RAG驱动的选股-组合-策略一体化引擎

**版本**: V3  
**日期**: 2026-06-03  
**核心变更**: 从"策略池驱动"转向"选股驱动+策略后置+RAG全链路监督"

---

## 一、核心理念

### 定位转变
- **旧思路**: 先建策略池(150个)→策略匹配标的→构建组合
- **新思路**: 先根据人和市场选股→为已选股票定制策略→RAG全程监督把关

### RAG角色定位
- **不是**: 前端生成策略的AI、用户可见的聊天机器人
- **而是**: 引擎内部的"投资顾问专家"，拥有否决权，效果不好时阻止进程并微调
- **输出对象**: 引擎内部模块，用户不可见
- **用户感知**: "正在优化中..."的进度提示

---

## 二、整体五步法

```
Step 1: 画像建立
  输入: 15题问卷
  输出: 数据A (15维向量 + 风险标签 + 经验标签)

Step 2: 市场信号捕捉
  输入: 五层信号采集
  输出: 数据B (市场状态向量)

Step 3: 静态选股器
  输入: A + B + 用户状态(资金/经验)
  输出: 数据C (50-100只候选股票)
  RAG监督: 因子有效性验证 → 不过关则微调重试

Step 4: 资产组合器
  输入: A + B + C
  输出: 数据D (组合配置)
  RAG监督: 配置合理性验证 → 不过关则微调重试

Step 5: 策略绑定器
  输入: D + B + A
  输出: 投资策略 (每只持仓的策略类型+参数)
  RAG监督: 策略匹配性验证 → 不过关则微调重试

持续: 寿命系统
  监控: 组合D
  输出: 寿命预警 + 调仓建议
```

## 三、各模块详细设计

### 3.1 静态选股器

**四阶段筛选**:

**阶段一: 硬过滤**
- 排除ST、退市风险
- 日均成交额>5000万
- 流通市值>50亿
- PE<100
- 数据源: akshare全市场扫描

**阶段二: 画像匹配**
- 保守型 → ATR<30%分位
- 稳健型 → ATR<50%分位
- 积极/激进型 → 放宽限制
- 小资金排除高价股

**阶段三: 周期匹配**
- 复苏期 → 偏好成长股、小市值
- 扩张期 → 偏好周期股、动量股
- 顶部期 → 偏好防御股、高股息
- 收缩期 → 偏好低波动、必需品

**阶段四: 多因子评分**
- 质量因子(ROE/毛利率/净利率)
- 估值因子(PE/PB/股息率)
- 动量因子(20日/60日/120日收益)
- 低波动因子(ATR/贝塔/波动率)
- 因子权重根据周期动态调整

**输出**: 50-100只候选股票 + 每只因子得分

---

### 3.2 资产组合器

**四个子步骤**:

**子步骤一: SAA战略资产配置**
- 输入: 画像风险等级 + 市场周期 + 宏观评分 + 地缘风险
- 输出: 股票/债券/商品/现金比例
- 例: 稳健型+复苏期+低风险 → 股票50% 债券35% 商品10% 现金5%

**子步骤二: TAA行业配置**
- 输入: 行业景气度 + 社会趋势 + 内部信号
- 输出: 各行业目标权重
- 例: 科技景气85分+AI趋势强 → 科技占股票部分的40%

**子步骤三: 个股精选**
- 从50-100只候选中按行业配额精选20-30只
- 每行业至少2只
- 优先综合因子得分高的

**子步骤四: 权重优化**
- 小白/新手 → 等权
- 有经验用户 → 风险平价

**输出**: 组合配置(持仓列表+权重+预期收益/风险)

---

### 3.3 策略绑定器

**三个子步骤**:

**子步骤一: 持仓特征分析**
- 提取特征标签: 波动率分位/趋势强度/估值水平/行业属性/市值大小

**子步骤二: 策略类型匹配**
- 高波动+强趋势+大盘 → 趋势跟踪
- 低波动+弱趋势+蓝筹 → 均值回归
- 低波动+无趋势+高股息 → 长期持有
- 高波动+小盘 → 严格止损+小仓位

**子步骤三: 参数优化**
- 贝叶斯优化或网格搜索
- 优化目标: 夏普比率 × 胜率 × 寿命预期

**输出**: 每只持仓的{策略类型+参数+买卖规则+止损线}

---

### 3.4 寿命系统

**三个维度**:

**维度一: 配置有效性寿命**
- 当前市场周期与组合配置匹配度
- 周期切换概率>50% → 寿命缩短

**维度二: 选股新鲜度**
- 距离上次选股时间
- 持仓股票基本面指标漂移程度
- 财报季更新后重新评分

**维度三: 策略有效性**
- 近期信号胜率
- 市场状态与策略适应度
- 胜率<40% → 寿命缩短

**组合寿命公式**:
```
组合寿命 = min(配置寿命×0.4, 选股寿命×0.3, 策略寿命×0.3)
```

**动态更新**:
- 每月1日自动运行
- 波动率加速因子: 高波动→×1.3 低波动→×0.85
- 周期适配因子: 匹配→1.0 相邻→0.9 不适→0.7
- 拥挤度因子: 家族超额收益高→×0.75

**预警阈值**:
- 寿命<6月: 黄色提醒
- 寿命<3月: 红色预警，推送替代方案

## 四、RAG全链路监督架构

### 4.1 RAG角色定位

- **角色**: 投资顾问专家，拥有否决权
- **输出对象**: 引擎内部模块(用户不可见)
- **工作方式**: 执行→评估→不过关→微调→重试→再评估(循环)
- **用户感知**: "正在优化中..."进度提示

### 4.2 三个优化循环

**循环一: 选股优化循环**
```
选股器→C→评估器→RAG判断
            ↑        ↓ 不过关
            └────────┘ RAG微调因子/规则
                      ↓ 过关
                   进入组合构建
```

**循环二: 组合优化循环**
```
组合器→D→评估器→RAG判断
            ↑        ↓ 不过关
            └────────┘ RAG微调权重/行业
                      ↓ 过关
                   进入策略绑定
```

**循环三: 策略优化循环**
```
策略绑定器→S→评估器→RAG判断
                  ↑        ↓ 不过关
                  └────────┘ RAG换策略类型/调参数
                            ↓ 过关
                         输出最终方案
```

### 4.3 RAG微调指令形式

RAG输出结构化微调指令，直接驱动引擎重跑:

**选股阶段指令示例**:
```json
{
    "adjustment_type": "factor_weight",
    "target": "static_stock_picker",
    "changes": {
        "momentum_weight": {"from": 0.35, "to": 0.25},
        "value_weight": {"from": 0.20, "to": 0.30}
    },
    "reason": "动量因子权重过高导致科技股过度集中",
    "expected_fix": "科技占比从45%降至30%"
}
```

**组合阶段指令示例**:
```json
{
    "adjustment_type": "weight_cap",
    "target": "portfolio_builder",
    "changes": {
        "single_stock_cap": {"from": 0.20, "to": 0.15}
    },
    "reason": "单只集中度超标",
    "expected_fix": "最大单只权重降至15%"
}
```

**策略阶段指令示例**:
```json
{
    "adjustment_type": "strategy_replace",
    "target": "strategy_binder",
    "changes": {
        "300750": {"from": "mean_reversion", "to": "trend_following"}
    },
    "reason": "高波动成长股配均值回归策略错配",
    "expected_fix": "策略-特征匹配度从50%提升至80%"
}
```

## 五、硬指标评估体系

### 5.1 选股阶段硬指标

| 指标 | 门槛 |
|------|------|
| 股票数量 | ≥30只 |
| 行业集中度 | 最大行业≤25% |
| 子行业隐性集中 | 同一子行业≤20% |
| 质量门槛 | 平均ROE≥10% |
| 估值合理性 | 平均PE<50 |
| 波动率匹配 | 与画像风险等级匹配 |
| 流动性 | 日均成交额>5000万 |
| 因子有效性 | 综合因子得分前30% |

### 5.2 组合阶段硬指标

| 指标 | 门槛 |
|------|------|
| 权重和 | =100% |
| 单只上限 | 保守10%/稳健15%/积极20%/激进25% |
| 行业上限 | ≤30% |
| 预期波动率 | <用户承受阈值 |
| 预期最大回撤 | <用户回撤阈值 |
| 夏普比率 | >0.3 |
| 压力测试 | 2022熊市回撤<用户阈值+5% |

### 5.3 策略阶段硬指标

| 指标 | 门槛 |
|------|------|
| 策略-特征匹配度 | >70% |
| 回测胜率 | >40% |
| 回测夏普 | >0.3 |
| 参数合理性 | 在论文验证范围内 |
| 止损合理性 | 2%<止损<15% |
| 策略多样性 | 组合内≥3种策略类型 |
| 同质化风险 | <0.6 |

### 5.4 相似经济周期回测(核心硬指标)

**方法**:
1. 识别当前市场状态(周期/波动率/情绪)
2. 在历史中找到最相似的N个时间段
3. 将当前方案"穿越"到历史情景
4. 计算历史表现
5. 综合评分 = 平均表现 × 稳定性

**通过标准**:
- 相似周期回测夏普 > 0.3
- 最差情景回撤 < 用户阈值 + 5%
- 表现稳定性(变异系数) < 0.5

## 七、RAG知识库动态更新

### 7.1 更新触发条件

| 触发源 | 更新内容 | 频率 |
|--------|---------|------|
| 市场信号检测 | 周期切换/波动率剧变/结构变化 | 实时 |
| 论文库新增 | 新论文结论/参数范围/有效性数据 | 每月 |
| 策略回测结果 | 实际表现vs预期/失效模式 | 每周 |
| 用户反馈 | 组合表现/调仓效果 | 每季度 |
| 人工审核 | 专家修正/规则更新 | 每季度 |

### 7.2 市场结构变化检测

信号检测机制感知变化，RAG自动调整:

**示例: 注册制改革全面实施**
- 小市值因子有效性下降 → 降低小市值暴露权重
- IPO增多壳价值消失 → 剔除壳价值相关策略
- 退市常态化 → 加强退市风险过滤

**RAG自动调整**:
- 选股器: 盈利能力门槛提升(ROE>10%→>15%)
- 组合器: 降低小盘股权重上限
- 策略器: 剔除依赖小市值溢价的策略模板

## 八、学习机制

RAG的"经验积累":

```json
{
    "scenario": "复苏期+稳健型+高动量权重",
    "problem": "科技占比45%过高",
    "adjustment": "动量权重35%→25%，价值权重20%→30%",
    "result": "成功，科技占比降至28%",
    "cycles": 2
}
```

积累后形成模式:
"复苏期+动量权重>30% → 大概率科技股集中 → 建议降低动量+提高价值"

下次相似场景直接应用。
