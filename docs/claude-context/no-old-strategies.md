# 项目准则：禁止使用现有低质量策略

**日期**: 2026-06-02
**来源**: 用户明确指示

## 准则内容

1. **现有策略（10-20个）均为低质量策略**，不得用于构建策略池和知识库
2. 策略池和知识库必须从**零开始构建**，所有模板需经过严格的回测验证（10股×3段×胜率>40%）才能入库
3. 如对新项目无影响，**可直接删除**现有低质量策略文件
4. 新策略入库标准：
   - 回测验证：10只不同类型股票 × 每段≥6个月 × 3个不同时间段
   - 胜率：每只、每段皆 > 40%
   - 年化超额：> 沪深300同期 + 2%
   - 最大回撤：< 25%
   - 夏普比率：> 0.5

## 影响范围

- `backend/app/seed_strategies/` 目录下的策略文件
- 数据库中已有的策略记录（如有）
- 所有与新策略池无关的历史策略代码

## 执行方式

- Phase B 策略模板全部新建，不引用/复用任何现有策略代码
- 论文知识库中的策略原理需重新代码化，不基于现有实现
