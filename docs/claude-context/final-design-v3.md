# 最终设计方案 V3（统一信号融合框架）

## 一、核心架构

### 策略表示层（统一接口）
所有策略（经典/ML/因子/套利/波动率）输出标准化信号：
```python
{
    "symbol": "510300",
    "date": "2026-06-01",
    "signal": 1,              # -1(卖) / 0(持有) / 1(买)
    "confidence": 0.75,       # 置信度 0-1
    "expected_return": 0.05,  # 预期收益
    "risk_estimate": 0.03     # 风险估计
}
```

### Hybrid引擎：三层融合

```
Layer 1: 策略池（各类型策略独立运行）
├── 经典策略 → 标准化信号
├── ML策略 → 标准化信号
├── 因子策略 → 标准化信号
├── 套利策略 → 标准化信号
└── 波动率策略 → 标准化信号

Layer 2: 信号融合（动态权重）
输入: 各策略信号 + 市场状态 + 用户画像
输出: 融合后的组合信号

融合方式:
├── 静态权重: 按夏普比率加权
├── 动态权重: 近期表现好的策略自动增权
└── 状态依赖: 高波动期增配波动率策略信号

Layer 3: 风控过滤
├── 组合信号置信度 < 0.5 → 不交易
├── 组合风险 > 用户承受 → 减仓
└── 同质化检查 → 避免信号来源过度集中
```

## 三、问卷设计（15题）

### 基本信息（5题）
1. 资金规模: <5万 / 5-20万 / 20-100万 / >100万
2. 年龄区间: 18-25 / 26-35 / 36-45 / 46-55 / 55+
3. 投资经验: 多选（股票/基金/债券/理财/都没买过）
4. 月度可支配收入占比: <10% / 10-30% / 30-50% / >50%
5. 是否有大额负债: 有压力大 / 有能承受 / 无

### 情景心理题（10题）
1. 损失厌恶: 1万跌到8000怎么办
2. 从众倾向: 朋友推荐跟不跟
3. 过度自信: 连涨3个月归因
4. 延迟满足: 2万奖金怎么用
5. 锚定效应: 解套心理
6. 资金安全感: 急用钱怎么办
7. 时间感知: 多久看不到收益能接受
8. 信息处理方式: 央行降准反应
9. 社会压力: 家人反对怎么办
10. 突发亏损应对: 一周跌20%

输出: 15维profile_vector

## 七、策略发现机制

Agent Crawler（每周日凌晨2:00）:
├── 爬取: GitHub Trending / arXiv q-fin.TR / 雪球 / 量化社区
├── 筛选: 代码可获取 / 有回测 / 近半年更新 / Star>50
├── 安全扫描 + 语法适配（迁移到akquant）
├── 回测验证: 10只不同类型股票 × 6个月 × 3段
├── DNA测序: 健康度>60
├── 寿命预测: >6个月
├── 同质化检查: 综合风险<0.7
└── 入池: 通过 → active / 未通过 → 记录原因

## 八、动态寿命机制

```
每月1日自动更新:
动态寿命 = 基础寿命 × 波动率加速因子 × 周期适配因子 × 拥挤度因子

波动率加速因子:
  波动率百分位 >150% → 1.3 / >120% → 1.15 / <50% → 0.85

周期适配因子:
  当前周期∈适应周期 → 1.0 / 相邻 → 0.9 / 不适 → 0.7

拥挤度因子(sigmoid):
  家族超额收益 >10% → 0.75 / >5% → 0.9 / <-5% → 1.1

组合层面:
  组合寿命 = min(各组件寿命)
  组合健康度 = 加权平均(各组件健康度)

再平衡触发（5种）:
  定期 / 偏离度 / 寿命预警 / 健康度下降 / 周期切换
```

## 九、已知遗留问题

### 问题1: 链路四直观化（用户要求后续提醒考虑）
当前链路四（全链路服务）设计较死板，用户希望有更直观的方法。
状态: 待后续讨论

### 问题2: 策略发现的LLM辅助
Agent Crawler纯人工/规则筛选效率低，是否需要LLM辅助代码理解/迁移？
建议: Phase B先手工启动，Phase C评估后决定是否引入LLM

### 问题3: 统一信号融合框架的潜在风险
- 不同策略类型的信号置信度不可比（ML的0.8 vs 经典的0.8含义不同）
- 信号融合可能放大系统性风险（所有策略同时失效时）
- 动态权重调整可能导致追涨杀跌
状态: 需Phase B验证后评估

### 问题4: 非经典策略的回测一致性
ML策略需要训练数据，回测时存在" lookahead bias "风险
套利策略需要双标的数据，回测框架需特殊处理
波动率策略需要更高频数据
状态: 需单独设计回测适配器

### 问题5: 样本量不足
初期策略池150个，某些画像×市场条件下的样本可能不足10个
状态: 初期用规则驱动，成熟期转数据驱动
