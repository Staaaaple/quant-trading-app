# Phase B 调整版实现计划 V2

## 核心调整

**原方案问题**: Phase B 做简化版（硬编码规则），Phase C 建知识库后推倒重来 → **返工风险大**

**新方案**: Phase B 直接包含**知识库基础建设** + **完整Hybrid 6步链路**，Phase C 只做**增量扩展**

### 2. 知识库基础（Week 2-3）

#### 2.1 策略模板库（七大策略家族 × 5个 = 35个）

| 家族 | 5个核心模板 |
|------|------------|
| 经典技术指标 | 双均线交叉、MACD背离、RSI超买超卖、布林带突破、量价共振 |
| 机器学习 | 随机森林择时、XGBoost分类、LSTM预测、Transformer时序、CNN模式识别 |
| 因子挖掘 | Fama-French五因子、Barra CNE5、舆情情感因子、资金流因子、分析师预期因子 |
| 统计套利 | 协整配对交易、距离法配对、跨期套利、产业链套利、贵金属套利 |
| 波动率交易 | GARCH预测、波动率择时、波动率期限结构、隐含vs实际波动率、VIX策略 |
| 行为金融 | 短期反转、长期动量、情绪指标合成、拥挤度指标、散户反向指标 |
| 事件驱动 | 财报超预期、分红送转、指数成分调整、解禁预警、政策利好 |

**每个模板包含**:
- Python代码（akquant框架适配）
- 参数搜索空间（JSON）
- 适用周期标注（复苏/过热/滞胀/衰退）
- 风险等级标注（保守/稳健/积极/激进）
- 回测验证结果（10只股票 × 6个月 × 3段）

#### 2.2 论文结构化数据库（20篇核心）

| 内容 | 结构化字段 |
|------|-----------|
| 论文元数据 | 标题、作者、发表时间、arXiv ID |
| 核心结论 | 策略原理简述、关键发现 |
| 参数范围 | 最优参数区间、搜索空间 |
| 市场状态映射 | 适用周期、适用市场类型 |
| 回测结果 | 夏普比率、最大回撤、胜率区间 |

**存储**: `models/paper_knowledge.py` + `services/paper_knowledge_service.py`

### 3. 完整Hybrid引擎（Week 3-4）

```
profile_vector × market_state_vector × pool_state_vector
    │
    ▼
Step 0: 策略池筛选（风险/周期/寿命硬过滤）
    │
    ▼
Step 1: SAA（战略资产配置）
    输入: profile.risk + market.macro + market.geo
    规则: 画像风险等级 × 市场周期 → 资产类别权重
    来源: 硬编码规则（Phase C替换为知识库驱动）
    │
    ▼
Step 2: TAA（战术资产配置）
    输入: market.industry + market.social + market.internal
    规则: 行业景气度 × 社会趋势 → 细分权重调整
    来源: 硬编码规则（Phase C替换为知识库驱动）
    │
    ▼
Step 3: 策略-标的绑定（贝叶斯优化）
    输入: 候选标的 + 策略池（35个模板）
    目标: max(夏普 × 健康度 × 寿命 × 标的相关性)
    约束: 风险≤用户承受 / 适合周期∩当前周期 / 寿命≥3月
    工具: Optuna/TPE贝叶斯优化
    │
    ▼
Step 4: 风控配置自动生成
    输入: profile_vector
    输出: 止损线、仓位上限、再平衡触发、最大回撤硬止损
    来源: 纯规则驱动（画像→风控参数映射）
    │
    ▼
Step 5: 可靠性评估
    - 组合回测（近3年）
    - 压力测试（2022熊市、2020疫情）
    - 蒙特卡洛模拟（1000次随机路径）
    │
    ▼
Step 6: 组合寿命计算
    组合寿命 = min(各组件寿命)
    组合健康度 = 加权平均(各组件健康度)
```

**关键设计**:
- SAA/TAA 先用硬编码规则，接口预留知识库注入点
- 策略绑定用真实35个模板，不是mock
- 贝叶斯优化是真实运行，不是伪代码

### 4. 回测适配器（Week 4-5）

| 资产 | 回测方式 | 说明 |
|------|---------|------|
| 个股 | akquant原生 | 日频OHLCV |
| ETF | akquant原生 | 交易规则同股票 |
| 可转债 | akquant原生 | 交易规则同股票 |
| 公募基金 | `services/fund_backtest_engine.py` | 净值+申购赎回费+T+1确认 |
| 期货 | 暂不开放直接交易 | 通过商品期货ETF间接配置 |

**回测验证标准**:
- 10只不同类型股票（跨行业/市值/风格）
- 单段回测 ≥ 6个月
- 测试段数 ≥ 3个不同时间段
- 胜率 > 40%
- 最大回撤 < 25%

## 逻辑检查

### 依赖关系

```
市场信号采集（独立，无依赖）
    │
    ▼
知识库基础（35模板+20论文）
    │
    ├──→ Hybrid引擎 SAA/TAA（知识库驱动）
    │
    ├──→ Hybrid引擎 策略绑定（策略池驱动）
    │
    └──→ 回测适配器（策略代码驱动）
              │
              └──→ 可靠性评估（回测结果驱动）
```

### 是否有遗漏？

| 原设计中的模块 | Phase B是否包含 | 说明 |
|----------------|----------------|------|
| 五层信号采集 | ✅ | Week 1 |
| 知识库（论文+模板） | ✅ | Week 2-3 |
| Hybrid 6步链路 | ✅ | Week 3-4 |
| 回测适配器 | ✅ | Week 4-5 |
| 策略DNA测序 | ⚠️ 简化版 | 只测35个核心模板，不做3D同质化 |
| 策略寿命预测 | ⚠️ 简化版 | 基于规则计算，不做动态更新 |
| Agent Crawler | ❌ | Phase C |
| 全链路服务（教学/推送） | ❌ | Phase D |

### 返工风险评估

| 模块 | 返工风险 | 应对 |
|------|---------|------|
| Hybrid框架 | 低 | 一次到位，接口预留 |
| SAA/TAA规则 | 中 | 硬编码→知识库驱动，但接口不变 |
| 策略绑定 | 低 | 35个真实模板，Phase C只是增加候选集 |
| 回测框架 | 极低 | 框架不变，只增加资产类型支持 |
| 前端页面 | 极低 | 展示层不变，数据层丰富 |
