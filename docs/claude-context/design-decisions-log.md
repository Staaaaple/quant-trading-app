## 2026-06-02 爬虫与信号采集改进

### 用户意见
> "信号采集是从哪里采集的？"
> "爬虫如果爬不到就尝试其他爬虫方式或其他网站"
> "爬虫和信号采集是否有避免虚假信息的方法"

### 修改内容

#### 1. 多源爬虫架构
- **主源**：`akshare.stock_news_main_cx()`（财联社100条，稳定可靠）
- **备选1**：新浪财经（HTML解析）
- **备选2**：东方财富快讯
- **备选3**：腾讯财经
- **备选4**：36氪科技
- **降级策略**：任一源失败自动切换，不影响整体

#### 2. 虚假信息过滤（5层）
| 过滤层 | 机制 |
|--------|------|
| 标题党过滤 | 黑名单正则16个模式："震惊"、"重磅"、"崩了"等 |
| 来源可信度 | 财联社0.9 > 证券时报0.88 > 新浪财经0.75 > 雪球0.55 |
| 去重 | 标题前20字相似度去重，保留高可信度版本 |
| 时效性 | 只采集最近24小时 |
| 宏观数据隔离 | GDP/CPI/PMI/M2/利率只信任akshare官方接口，不通过新闻获取 |

#### 3. 信号来源透明化
- Layer 1 宏观：akshare官方数据（国家统计局/央行）
- Layer 2-4 新闻：财联社（akshare封装）
- Layer 5 内部：akshare（沪深300/国债收益率）

### 文件变更
- `backend/app/services/news_crawler.py`（完整重写）
- `backend/app/services/market_signal_service.py`（更新数据来源说明）
