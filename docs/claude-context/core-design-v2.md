# 核心设计决策 V2（2026-06-01）

## 一、产品定位：全链路智能投顾服务

不是"给策略自己玩"，而是"从画像到实操的全程陪伴"。

服务链路：
1. 完成画像 → 2. 生成定制组合+策略 → 3. 教学引导（怎么开户、怎么买）
→ 4. 建仓助手（分批计划+操作截图） → 5. 日常推送（有操作时才推送）
→ 6. 调仓提醒（策略到期/周期切换/寿命预警） → 7. 周报月报+教学卡片

**不推具体券商**，只说明挑选标准（费率、APP体验、研报质量）。
**定制在教学之后**，根据具体投资策略做引导和教学。

## 二、问卷设计（情景化+基本信息）

### 基本信息（3题，无压力）
1. 资金规模：单选 <5万 / 5-20万 / 20-100万 / >100万
2. 投资经验：是否买过股票/基金/债券/理财/都没买过
3. 收入来源：工资为主 / 生意收入 / 投资收入 / 其他

### 情景心理题（7题，行为金融学六维）
1. **损失厌恶**："1万基金跌到8000，你会？"（A立刻全卖 B卖一半 C不动 D加仓）
2. **从众倾向**："朋友说某股票涨了30%，你会？"（A马上跟买 B研究后决定 C不为所动 D觉得该卖了）
3. **过度自信**："连涨3个月赚20%，你觉得主要原因是？"（A我眼光好 B我研究了 C市场好 D运气）
4. **延迟满足**："突然收到2万奖金，你会？"（A全存银行 B大部分存小部分投 C一半投一半消费 D全投高风险）
5. **锚定效应**："10元买入跌到7元，涨回9元，你会？"（A解套就卖 B等10元卖 C看趋势 D该加仓）
6. **资金安全感**："明天突然需要3万急用，能立刻拿出来吗？"（A完全没问题 B要卖投资 C凑不齐 D没想过）
7. **时间感知**："能接受多久看不到明显收益？"（A1个月 B3个月 C1年 D3年以上）

### 画像计算（多维度结构化输出）
```python
# 输出为结构化向量，直接输入Hybrid引擎
profile_vector = {
    "risk_tolerance": 1-10,           # 综合风险承受
    "loss_aversion": 1-10,            # 损失厌恶（影响止损设置）
    "herding_tendency": 1-10,         # 从众倾向（影响是否需要逆向提醒）
    "overconfidence": 1-10,           # 过度自信（影响是否需要冷静期）
    "delayed_gratification": 1-10,    # 延迟满足（影响策略持有周期）
    "security_need": 1-10,            # 资金安全感（影响流动性配置）
    "time_horizon": short/medium/long, # 时间感知
    "experience_level": none/beginner/intermediate/advanced,
    "capital_tier": <50k/50k-200k/200k-1m/>1m,
}
```

## 四、Hybrid组合设计引擎（核心重构）

输入三向量：
- `profile_vector`（用户画像结构化输出）
- `market_state_vector`（市场五层信号加权输出）
- `pool_state_vector`（策略池状态：各策略健康度/寿命/同质化）

### 引擎架构

```
profile_vector × market_state_vector × pool_state_vector
    │
    ▼
┌─────────────────────────────────────────────────────────────┐
│  Hybrid Portfolio Designer                                  │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  Step 1: 战略资产配置 (SAA)                                  │
│  输入: profile.risk_tolerance + market.macro + market.geopolitical│
│  规则:                                                       │
│    高风险厌恶 + 高地缘风险 → 股票权重下调10%，黄金上调10%       │
│    积极型 + 复苏期 + 低风险 → 股票上限70%                     │
│    保守型 + 任何周期 → 股票上限30%，债券下限50%               │
│  输出: 资产类别权重 {stock: 0.4, bond: 0.4, commodity: 0.1, cash: 0.1}│
│                                                             │
│  Step 2: 战术资产配置 (TAA)                                  │
│  输入: market.industry + market.social + market.market_internal│
│  规则:                                                       │
│    科技景气85分 + AI社会趋势90分 → 科技ETF在股票中占50%        │
│    消费降级趋势 → 减配高端消费，增配平价消费                  │
│    股债利差>4% → 增配股票                                    │
│    北向资金连续流出 → 减配白马蓝筹                            │
│  输出: 资产内部细分权重                                       │
│                                                             │
│  Step 3: 策略-标的绑定 (贝叶斯优化)                          │
│  输入: 候选标的 + 策略池状态                                  │
│  目标函数: max(策略历史夏普 × 策略健康度 × 策略寿命月数 × 标的相关性)│
│  约束:                                                       │
│    - 策略风险等级 ≤ 用户风险承受                              │
│    - 策略适合周期 ∩ 当前周期 ≠ ∅                            │
│    - 策略寿命 ≥ 3个月                                        │
│    - 组合同质化风险 < 0.6                                    │
│  输出: 每个标的的最优策略绑定                                 │
│                                                             │
│  Step 4: 风控配置自动生成                                    │
│  根据画像.loss_aversion自动设置:                              │
│    - 止损线: 高损失厌恶 → 5%止损 / 低 → 15%                  │
│    - 仓位上限: 根据风险承受动态计算                           │
│    - 再平衡触发: 偏离度5%（保守）/ 10%（积极）                │
│    - 最大回撤硬止损: 画像决定（保守10% / 积极25%）            │
│                                                             │
│  Step 5: 可靠性评估                                          │
│  - 组合回测: 近3年历史表现                                    │
│  - 压力测试: 2022年熊市、2020年疫情等极端场景表现               │
│  - 蒙特卡洛模拟: 1000次随机路径，计算收益分布                  │
│  - 输出: 预期年化收益、95%置信区间最大回撤、胜率               │
│                                                             │
│  Step 6: 组合寿命计算                                        │
│  - 组合寿命 = min(各组件策略寿命)                            │
│  - 组合健康度 = 加权平均(各组件健康度)                        │
│  - 输出: 组合预期寿命、健康度评分                             │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 五、策略发现机制（替代原链路3）

**原链路3丢弃**，改为**智能体策略发现**：

```
定时任务（每周日凌晨2:00）
    │
    ▼
Agent Crawler:
├── 爬取目标:
│   ├── GitHub Trending (quant trading repos)
│   ├── arXiv最新量化论文 (q-fin.TR)
│   ├── 雪球热门策略帖
│   ├── 知乎/掘金量化专栏
│   └── 知名量化博客 (如优矿、聚宽社区)
│
├── 筛选规则:
│   ├── 代码可获取（有开源实现）
│   ├── 有回测结果展示
│   ├── 近半年有更新（非废弃项目）
│   └── Star数>50 或 社区讨论度较高
│
├── 验证流程:
│   ├── 代码安全扫描（无恶意代码）
│   ├── 语法适配（迁移到akquant框架）
│   ├── 回测验证（10只不同类型股票×6个月×3段）
│   ├── DNA测序（健康度>60）
│   ├── 寿命预测（>6个月）
│   └── 同质化检查（综合风险<0.7）
│
└── 入池:
    ├── 通过全部验证 → 进入策略池（状态: active）
    └── 未通过 → 记录失败原因，供人工review
```
