## 一、项目定位

**QuantEvo** 是一个面向投资小白的智能量化投资助手，核心目标是：

> **让零基础用户通过15题问卷，5分钟内获得一套完整的、经过RAG+LLM质检的投资组合。**

从专业量化工具转型后，当前聚焦：
- 简化用户上手路径（15题问卷 → 18维画像）
- 动态实时选股（替代静态映射）
- RAG+LLM全程质检（6步循环审核）
- 全链路投后服务（建仓/推送/再平衡/周报）

│  ├── 技术面: EMA/量能/波动率/RSI/MACD                       │
│  │   来源: akshare.stock_zh_a_hist                           │

    │
    ▼
┌─────────────────────────────────────────────────────────────┐

│  ├── 基本面过滤: ROE≥8%, 毛利率≥15%, 净利率≥5%              │
│  ├── 技术面过滤: EMA趋势健康, 波动率适中, 排除暴涨           │
│  ├── 资金面过滤: 主力净流入≥-1亿, 融资变化≥-5%              │
│  └── 控盘度过滤: 机构持仓≥15%, 股东户数变化≤15%             │

    │
    ▼
┌─────────────────────────────────────────────────────────────┐
│ Step 4: RAGLLMSelector 精选                                  │
│  ├── 构建标的画像 (多维度数据摘要)                           │
│  ├── RAG检索: 相似历史案例/行业研报/公司公告                  │
│  ├── LLM评估: 综合评分 + 选股理由 + 风险提示                  │

    │

### 3.3 Hybrid引擎RAG质检流程（6步循环审核，每步最多20次）

RAG投资顾问**不对外暴露**，而是嵌入Hybrid组合引擎的6步中，每步输出作为RAG输入，RAG审核结果用于调整该步参数并重试。**所有步骤统一最多循环20次**。

│  │ 计算逻辑: 风险画像硬约束 × 市场周期调整系数 × 宏观/地缘微调 → 归一化 │   │

### 动态选股的增强RAG调试（Step 3内部机制）

动态选股（Step 3）的RAG质检是**增强版**，与其他步骤不同：

```
标准RAG质检（Step 1/2/4/5/6）:
  RAG检索 → LLM评估 → adjustments → 应用调整 → 重试该步骤

增强RAG质检（Step 3 动态选股）:
  RAG检索 → LLM评估 → 选股失败分析
    │
    ▼ LLM分析失败根因
  ┌─────────────────────────────────────┐
  │ 1. 标的池太小？ → 扩大pool_size     │
  │ 2. 筛选太严格？ → 放宽阈值          │
  │ 3. 目标太高？ → 降低select_count    │
  │ 4. 某行业无标的？ → 放宽该行业条件  │
  │ 5. 数据获取失败？ → 调整数据源      │
  └─────────────────────────────────────┘
    │
    ▼ 应用调整后的参数
  完整重跑动态选股4步流程 (Pool→Enrich→Filter→RAGLLMSelect)
    │
    ▼ 再次RAG质检
  通过 → 进入Step 4
  失败 → 继续分析调整（与其他步骤统一，最多20次）
```

**关键区别：**
- 标准步骤：RAG只审核结果，调整是简单的权重/参数修改，重试该步骤
- 动态选股：RAG不仅审核结果，还**分析失败根因**，LLM**智能调整选股策略参数**，然后**完整重跑选股4步流程**
- 动态选股的RAG调试是**递归式**的（选股内部有LLM调整 + 外部有RAG质检），但**循环次数与其他步骤统一为20次**

## 十一、策略寿命系统（详细设计）

**状态**: 核心算法已实现，前端展示已搭建，自动化监控待完善

**目的**: 解决"策略什么时候失效"的核心问题

**生物学隐喻**:

```
策略 = 生物个体
├─ DNA: 策略的底层逻辑（动量/均值回归/趋势跟踪）
├─ 代谢率: 策略对数据新鲜度的敏感程度（高频=高代谢=短寿命）
├─ 生态位: 策略适应的市场环境（牛市/熊市/震荡）
├─ 寿命: 策略边缘的有效期（月）
└─ 健康度: 策略当前运行状态（0-100）
```

**寿命计算公式**:

```python
def calculate_lifespan(metabolic_rate, niche_width, metabolic_syndrome, homogeneity_risk):
    # 基础寿命: 代谢率越高 = 需要越新鲜的数据 = 寿命越短
    base_lifespan = max(6, 60 - metabolic_rate * 120)
    
    # 生态位宽度加成: 适应场景越多 = 寿命越长
    niche_bonus = niche_width * 24
    
    # 同质化惩罚: 太多人用 = 拥挤交易 = 寿命缩短
    homogeneity_penalty = homogeneity_risk * 20
    
    raw_lifespan = base_lifespan + niche_bonus - homogeneity_penalty
    
    # 代谢综合征: 结构性脆弱 = 严重惩罚
    if metabolic_syndrome:
        raw_lifespan *= 0.6
    
    lifespan_months = max(1, round(raw_lifespan))
    
    # 寿命阶段
    if lifespan_months > 36:   phase = "年轻"
    elif lifespan_months >= 12: phase = "成熟"
    elif lifespan_months >= 3:  phase = "衰老"
    else:                       phase = "濒危"
    
    return {
        lifespan_months,
        phase,
        aging_velocity,  # 每月边缘衰减百分比
        recommendations, # 延寿建议
    }
```

**寿命影响因素**:

| 因素 | 说明 | 对寿命的影响 |
|------|------|------------|
| **代谢率** | 策略对数据新鲜度的敏感度 | 高代谢（高频）→ 短寿命 |
| **生态位宽度** | 策略适应的市场环境数量 | 宽生态位 → 长寿命 |
| **同质化风险** | 市场上相似策略的数量 | 高同质化 → 短寿命 |
| **代谢综合征** | 策略是否存在结构性缺陷 | 有 → 寿命 × 0.6 |
| **波动率** | 市场波动程度 | 高波动 → 加速衰老 |
| **周期适配** | 策略与当前周期的匹配度 | 不匹配 → 加速衰老 |

**动态监控流程（规划中）**:

```
每月1日凌晨3:00 定时任务
    │
    ├──→ 采集市场环境数据
    │   ├── 波动率20日ATR / 历史波动率百分位
    │   ├── 当前周期阶段
    │   └── 家族拥挤度（近3月超额收益）
    │
    ├──→ 遍历所有活跃策略
    │   ├── 基础寿命（静态计算）
    │   ├── × 波动率加速因子（高波1.3 / 低波0.85）
    │   ├── × 周期适配因子（匹配1.0 / 相邻0.9 / 不适0.7）
    │   └── × 拥挤度因子（sigmoid映射超额收益）
    │
    ├──→ 计算组合层面寿命
    │   ├── 组合寿命 = min(各组件寿命)
    │   └── 组合健康度 = 加权平均
    │
    ├──→ 寿命变化检测
    │   ├── 减少>20% → 黄色预警
    │   └── 减少>40% 或 <3月 → 红色预警
    │
    ├──→ 写入寿命历史表
    │
    └──→ 推送预警到Dashboard
```

**前端展示**:

- **策略匹配页**: 每个策略显示寿命进度条（早期绿/中期橙/后期红）
- **寿命监控页**: 所有策略的寿命总览，濒危策略高亮
- **Dashboard**: 组合总寿命 + 健康度评分

**策略有效性底线（硬约束）**:

```
任何策略必须满足：策略收益 ≥ 买入持有收益（Buy-and-Hold Benchmark）

买入持有基准定义：
├─ 股票部分：买入并持有沪深300ETF(510300)，同期收益
├─ 债券部分：买入并持有国债ETF(511010)，同期收益
├─ 商品部分：买入并持有黄金ETF(518880)，同期收益
├─ 现金部分：买入并持有银华日利(511880)，同期收益
└─ 组合基准 = Σ(资产权重 × 对应资产买入持有收益)

检查维度：
├─ 全周期累计：策略累计收益 ≥ 基准累计收益（硬约束，一票否决）
├─ 分年度检查：任一年度策略收益 ≥ 基准收益 × 0.9（防止单年大幅跑输）
├─ 超额收益α：剔除β后纯策略收益 ≥ 0（确保不是市场β的功劳）
├─ 超额稳定性：超额收益夏普比率 ≥ 0.3（确保超额不是偶然）
└─ 最大相对回撤：策略相对基准的最大回撤 ≤ 15%（防止阶段性跑输过多）

未通过处理：
├─ Step 5 未通过 → 调参/换策略/调权重，重新回测
├─ Step 6 未通过 → 返回Step 1~5重试，或标记为"观察期策略"
└─ 连续3次未通过 → 淘汰该策略模板，从策略池中移除
```

| 决策 | 选择 | 原因 |

## 十二、RAG质检 vs 动态选股RAG的区别

| 维度 | 标准RAG质检 (Step 1/2/4/5/6) | 增强RAG质检 (Step 3 动态选股) |
|------|------------------------------|------------------------------|
| **输入** | 该步骤的输出结果 | 选股全流程结果 + 中间状态 |
| **RAG检索** | 该步骤相关理论+案例 | 历史绑定案例+行业研报+公司公告+选股策略 |
| **LLM评估** | 结果合理性 | 标的质量+回测表现+**失败根因分析** |
| **调整方式** | 简单参数修改 | **智能策略调整** (扩大池/放宽阈值/降低目标) |
| **重试范围** | 该步骤重新计算 | **完整重跑选股4步流程** |
| **递归深度** | 单层循环 | **双层递归** (选股内部LLM调整 + 外部RAG质检) |
| **最大循环** | **20次** | **20次** (与其他步骤统一) |
