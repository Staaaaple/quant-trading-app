# QuantEvo 策略生态系统详解

QuantEvo 的策略生态系统（Strategy Ecosystem）是产品最核心的差异化能力之一。它把每个交易策略都视为一个**活的有机体**，从基因构成、家族关系、新陈代谢到生命周期进行全方位刻画，帮助用户识别策略失效风险、避免同质化陷阱，并持续优化策略群落。

---

## 一、核心设计理念

传统量化平台只回答“策略过去赚了多少”，QuantEvo 更关心“策略是什么”“它会活多久”“它和谁太像”。

策略生态系统的生物隐喻包含四个层次：

| 生物学概念 | 策略学含义 | 解决的问题 |
|-----------|-----------|-----------|
| DNA | 策略的基因结构 | 策略到底依赖什么因子、什么逻辑 |
| 系统发育（Phylogeny） | 策略家族与亲缘关系 | 策略是否过度相似、是否存在同质化风险 |
| 代谢（Metabolism） | 策略对数据环境的依赖程度 | 策略是否太娇贵、是否容易因市场变化而失效 |
| 寿命（Lifespan） | 策略的有效剩余时间 | 什么时候该替换策略 |

## 二、DNA 测序：解析策略的基因结构

### 2.1 四层基因检测

QuantEvo 通过**关键词匹配 + AST 静态解析**，自动从策略代码中提取四层基因：

| 层级 | 检测内容 | 典型基因标签 |
|------|---------|-------------|
| **特征层（Feature）** | 技术指标 / 基本面因子 | MA、MACD、RSI、布林带、成交量、PE、PB、ROE、ATR |
| **信号层（Signal）** | 交易逻辑类型 | 趋势跟踪、均值回归、动量突破、规则驱动、机器学习 |
| **风控层（Risk）** | 风险控制机制 | 固定止损、ATR 止损、跟踪止损、仓位上限、最大回撤限制 |
| **执行层（Execution）** | 交易执行方式 | 市价单、限价单、TWAP、滑点控制 |

AST 解析会进一步修正关键词匹配结果。例如，如果代码中导入了 sklearn、PyTorch 或 XGBoost，信号层会自动添加 `ml_model`；如果代码包含类定义，会强化 `rule_based` 标签。

### 2.2 32 维基因向量

四层基因会被编码为一个固定的 **32 维二进制向量**（每层 8 维），作为策略的“数字身份证”。这个向量后续用于计算策略相似度、家族聚类和同质化风险。

### 2.3 关键输出指标

- **基因多样性分数（gene_diversity_score）**：四层唯一基因总数 / 16，反映策略结构的丰富程度。
- **健康出生分数（health_birth_score）**：基础分 50 + 多样性奖励 + 风控奖励 + ML 奖励，综合评估策略“先天健康度”。
- **基因画像（gene_profile）**：完整的四层标签集合与 AST 特征。

## 三、系统发育：构建策略家族树

### 3.1 相似度计算

任意两个策略的基因向量通过**余弦相似度**计算亲缘关系，结果在 0 到 1 之间。1 表示几乎完全相同，0 表示毫无关联。

### 3.2 家族聚类

QuantEvo 预设了若干策略家族，例如：

- **趋势跟踪家族**：双均线、MACD、EMA、突破策略
- **动量家族**：RSI、KDJ、成交量、20 日动量
- **均值回归家族**：布林带、波动率策略
- **多因子家族**：PE/PB、ROE、小市值、价值动量
- **风控增强家族**：ATR、动态仓位

系统会根据策略 ID 关键词和基因向量自动将策略归入相应家族。

### 3.3 近亲繁殖预警

当某个策略与其最相似的 5 个“亲属”的平均相似度超过 **0.75** 时，系统会触发**近亲繁殖警告（inbreeding_warning）**，提示用户该策略可能与已有策略过度同质化，需要引入差异化逻辑。

## 四、代谢分析：衡量策略的环境依赖

### 4.1 信息代谢率（Metabolic Rate）

策略对数据更新频率和处理量的需求越高，代谢率越高：

| 频率 | 基础代谢率 |
|------|-----------|
| 分钟/高频 | 0.30 |
| 日频 | 0.10 |
| 周频 | 0.05 |
| 月频 | 0.02 |

代谢率还会受到复杂度因子和 ML 模型影响。代谢率越高，策略对近期数据越敏感，老化速度越快。

### 4.2 生态位宽度（Niche Width）

反映策略能适应的市场场景广度：
- 特征基因越多，基础宽度越大；
- 覆盖趋势、反转、价值、成交量等多种市场状态会获得额外奖励；
- 风控机制越完善，适应力越强。

### 4.3 代谢综合征（Metabolic Syndrome）\n
当策略存在结构性脆弱时，会被标记为代谢综合征：
- 特征基因少于 2 个 → “单一指标依赖”
- 信号基因少于 2 个 → “信号逻辑过于简单”
- 无风控基因 → “无风控机制”
- 纯规则驱动 → “缺乏自适应能力”

## 五、寿命预测：预判策略的剩余价值

### 5.1 寿命计算模型

策略剩余寿命（月）由以下因素综合决定：

```
基础寿命 = max(6, 60 - 代谢率 × 120)
生态位奖励 = 生态位宽度 × 24
同质化惩罚 = 同质化风险 × 20
原始寿命 = 基础寿命 + 生态位奖励 - 同质化惩罚
若存在代谢综合征：原始寿命 ×= 0.6
最终寿命 = max(1, round(原始寿命))
```

### 5.2 寿命阶段

| 剩余月数 | 阶段 | 含义 |
|---------|------|------|
| > 36 | 年轻期 | 策略健康，可长期持有 |
| 12–36 | 成熟期 | 策略表现稳定，需定期关注 |
| 3–12 | 衰老期 | 策略 edge 可能衰减，建议准备替代 |
| < 3 | 濒危期 | 策略即将失效，应立即审查或替换 |

### 5.3 老化速度

老化速度综合代谢率、同质化风险和代谢综合征：

```
老化速度 = 代谢率 × 0.5 + 同质化风险 × 0.3 + (代谢综合征 ? 0.1 : 0)
```

### 5.4 自动监控与替代推荐

系统每月自动运行寿命检查，结合市场环境因子（波动率、经济周期、拥挤度）动态调整。当策略濒危时，会自动推荐与其**相似度低于 0.8** 但健康度更高的替代策略，帮助用户平滑过渡。

## 六、生态仪表盘：一目了然的策略群落

前端生态仪表盘将上述指标可视化呈现：

- **家族分布**：各家族策略数量占比
- **健康度分布**：策略健康度区间分布
- **寿命分布**：策略剩余寿命区间分布
- **家族能力雷达**：健康度、多样性、代谢稳定性、生态位宽度、差异化五维对比
- **策略关系网络**：力导向图展示策略间的亲缘关系，连线越粗表示越相似
- **需关注列表**：濒危策略、低健康度策略、高代谢策略

---

## 七、典型应用场景

1. **策略入库**：新策略上传后自动完成 DNA 测序，并归入家族。
2. **组合审查**：查看组合中各策略的寿命分布，避免多个策略同时进入衰老期。
3. **同质化排查**：通过近亲繁殖预警发现过度相似策略，及时引入差异化逻辑。
4. **策略替换**：当某策略濒危时，系统推荐替代方案，降低组合失效风险。
