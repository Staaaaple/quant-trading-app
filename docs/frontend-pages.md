# QuantEvo 各页面在产品中的意义与作用

> 本文档从**产品应用角度**（而非技术实现角度）说明 QuantEvo 每个页面对用户的价值、在整个投资流程中的位置，以及它为后续功能提供了哪些关键输入。

---

## 一、首页：产品的门面与个人状态总览

**页面**：`/` → [HomeView.vue](frontend/src/views/HomeView.vue)

### 意义与作用
- **产品定位**：用户打开 QuantEvo 的第一屏，也是贯穿整个使用过程的“仪表盘 + 导航中枢”。
- **给用户什么**：
  - 未创建画像时：清晰告诉用户“这里是量化投资平台，先从画像开始”，降低冷启动焦虑。
  - 创建画像后：一眼看到当前市场状态、自己的组合收益、资产配置概况，以及进入画像、市场、组合、回测等模块的快捷入口。
- **为后续功能提供什么**：
  - 通过快捷入口把用户引导到下一个动作（去画像、看市场、生成组合、查看今日操作）。
  - 让复杂系统被拆成可感知的“状态卡片”，用户不需要理解后端算法也能知道今天该关注什么。

## 二、投资者画像：所有个性化推荐的“输入源头”

**页面**：
- `/profile` → [ProfileWizard.vue](frontend/src/views/ProfileWizard.vue)
- `/profile/summary` → [ProfileSummary.vue](frontend/src/views/ProfileSummary.vue)

### 意义与作用
- **产品定位**：把抽象的“你是什么样的投资者”变成系统可计算、可匹配的约束条件。后续所有推荐（组合、策略、建仓节奏）都基于画像结果。
- **给用户什么**：
  - 通过 15 道贴近真实场景的问题，让用户意识到自己的风险偏好、投资经验、收益预期和行为习惯。
  - 生成一个可理解的风格标签（如“稳健守护者”），并给出与其匹配的初步配置建议。
- **为后续功能提供什么关键输入**：
  - **投资组合生成**：画像决定 SAA（战略资产配置）的股票/债券/商品/现金比例。例如激进型用户股票比例更高，保守型用户债券和现金比例更高。
  - **策略匹配**：不同画像会匹配不同风险等级的策略。
  - **建仓引导**：风险承受能力影响建仓节奏（激进可以快一点，稳健需要分阶段）。
  - **周报/今日操作**：画像决定了同样的市场信号下，给用户的建议语气和风险提醒强度。

## 三、市场信号：判断“现在是不是好时机”

**页面**：
- `/market/guide` → [MarketGuideView.vue](frontend/src/views/MarketGuideView.vue)
- `/market` → [MarketSignalView.vue](frontend/src/views/MarketSignalView.vue)

### 意义与作用
- **产品定位**：回答用户最关心的问题之一——“现在市场环境怎么样？我该激进还是保守？”
- **给用户什么**：
  - 一个综合市场评分，把宏观、地缘、行业、情绪、内部流动性等复杂信息压缩成一目了然的分数。
  - 经济周期可视化，让用户知道自己处于“复苏/过热/滞胀/衰退”哪个阶段。
  - 每一层信号的解释说明，帮助用户理解分数背后的含义，而不是盲目相信一个数字。
- **为后续功能提供什么关键输入**：
  - **投资组合生成**：市场信号是 Hybrid 引擎做 TAA（战术资产配置调整）的重要依据。例如市场过热时可能会降低股票暴露、提高现金比例。
  - **今日操作**：市场简报直接引用市场信号的周期和综合评分，作为是否建议买入/卖出的宏观背景。
  - **周报**：市场回顾部分复用市场信号，解释本周组合表现的市场原因。

## 四、投资组合：把“你是谁”和“市场怎么样”变成具体配置

**页面**：
- `/portfolio/guide` → [PortfolioGuideView.vue](frontend/src/views/PortfolioGuideView.vue)
- `/portfolio` → [PortfolioBuilder.vue](frontend/src/views/PortfolioBuilder.vue)

### 意义与作用
- **产品定位**：QuantEvo 的核心输出页。把画像和市场信号转化为用户真正可以执行的资产配置方案。
- **给用户什么**：
  - 股票、债券、商品、现金四类资产的配置比例（SAA）。
  - 每类资产下具体推荐的标的（如沪深 300 ETF、国债 ETF、黄金 ETF、货币基金）及推荐理由。
  - 风险配置说明和购买指引，让用户知道“这个组合为什么适合我”以及“怎么买”。
- **为后续功能提供什么关键输入**：
  - **策略匹配**：组合中的每类资产需要对应到具体策略（股票用什么策略、债券用什么策略）。
  - **回测中心**：组合与策略绑定后，才能批量跑回测，验证这个配置在历史上表现如何。
  - **今日操作 / 周报**：组合绑定中的标的和回测结果，是生成每日买卖信号和每周绩效报告的数据基础。
  - **建仓引导**：组合结果决定“要买哪些、各买多少”，建仓引导把它拆成可执行的步骤。

## 五、策略匹配与执行：把资产配置翻译成可运行的策略

**页面**：
- `/portfolio/strategies/guide` → [StrategyGuideView.vue](frontend/src/views/StrategyGuideView.vue)
- `/portfolio/strategies` → [StrategyMatch.vue](frontend/src/views/StrategyMatch.vue)
- `/recommendation` → [StrategyRecommendation.vue](frontend/src/views/StrategyRecommendation.vue)

### 意义与作用
- **产品定位**：资产配置是“买什么”，策略匹配是“怎么买”。这一页把抽象的资产类别落到具体的交易策略上，并告诉用户如何执行。
- **给用户什么**：
  - 每类资产匹配的策略列表（如股票用趋势跟踪、债券用久期管理、商品用通胀对冲）。
  - 每个策略的预期收益、风险等级、夏普率、最大回撤等关键指标。
  - 策略生命周期状态（年轻/成熟/衰老/濒危），提醒用户策略是否还有效。
  - 实盘 API 对接说明（QMT、JoinQuant 等），让有交易能力的用户知道如何落地。
- **为后续功能提供什么关键输入**：
  - **回测中心**：策略被选中后，会进入回测队列，验证历史表现。
  - **A/B 测试**：用户可以对不同策略做对比实验。
  - **模拟盘监控**：策略可以放入模拟盘，在不投入真实资金的情况下观察实际运行效果。
  - **策略生态系统 / 基因报告**：策略进入生态系统后，会被持续分析健康度、寿命和家族关系。

## 六、策略生态系统：长期监控策略群落健康

**页面**：
- `/ecosystem` → [EcosystemView.vue](frontend/src/views/EcosystemView.vue)
- `/dna-report/:strategy_id` → [DNAReport.vue](frontend/src/views/DNAReport.vue)
- `/strategy-map` → [StrategyMapView.vue](frontend/src/views/StrategyMapView.vue)

### 意义与作用
- **产品定位**：QuantEvo 区别于普通量化平台的特色能力。把策略当作“活的有机体”，持续监测它们的健康、寿命、代谢和家族关系，防止策略同质化或失效。
- **给用户什么**：
  - **生态系统仪表盘**：策略总数、家族分布、健康分、濒危策略数量等宏观指标。
  - **关系网络图**：直观看到哪些策略基因相似、是否存在“近亲繁殖”。
  - **基因报告**：单个策略的 DNA 分析——特征层、信号层、风控层、执行层，以及寿命阶段和优化建议。
  - **策略地图**：预设策略库，用户可以浏览并导入优秀策略，降低策略创作门槛。
- **为后续功能提供什么关键输入**：
  - **策略推荐**：生态健康度会影响策略是否被推荐给用户。濒危或高近亲繁殖风险的策略会被降级。
  - **周报**：寿命预警会进入周报，提醒用户某些策略可能需要替换。
  - **A/B 测试 / 回测**：发现潜在问题策略后，可以发起 A/B 测试或重新回测，验证替代方案。

## 七、建仓引导：把组合建议变成一步一步的交易计划

**页面**：`/building-guide` → [BuildingGuide.vue](frontend/src/views/BuildingGuide.vue)

### 意义与作用
- **产品定位**：很多投资平台只告诉用户“该买什么”，但没说“怎么买”。建仓引导填补了“建议”到“真实持仓”之间的最后一公里。
- **给用户什么**：
  - 从开户、资金转入到分阶段建仓的 11 步清单。
  - 默认的建仓节奏（如 40% / 35% / 25%），帮助用户避免一次性重仓的心理压力。
  - 每一步的操作提示和检查框，进度保存在本地，用户可以分多次完成。
- **为后续功能提供什么关键输入**：
  - **今日操作**：建仓计划中的标的，后续会进入每日操作监控，提醒用户按计划买入。
  - **周报 / 模拟盘**：建仓完成后，用户可以在周报中追踪实际持仓表现，或在模拟盘中验证策略。

## 八、每日运营：让投资建议变成日常陪伴

**页面**：
- `/today-operation` → [frontend/src/views/TodayOperation.vue](frontend/src/views/TodayOperation.vue)
- `/weekly-report` → [frontend/src/views/WeeklyReport.vue](frontend/src/views/WeeklyReport.vue)

### 意义与作用
- **产品定位**：投资不是一次性决策，而是持续管理。今日操作和周报让平台从“工具”变成“日常陪伴”。
- **今日操作给用户什么**：
  - 当天的市场简报。
  - 基于组合回测结果生成的具体买入/卖出信号（标的、策略、置信度、建议金额）。
  - 用户可以对信号进行“确认”或“忽略”，形成互动反馈。
- **周报给用户什么**：
  - 本周组合收益、相对基准的超额收益、最大回撤。
  - 最佳/最差贡献标的，帮助用户理解收益来源。
  - 市场回顾与后市展望，以及基于画像的个性化建议。
  - 策略寿命预警，提醒用户哪些策略可能失效。
- **为后续功能提供什么关键输入**：
  - 用户的“确认/忽略”操作未来可用于优化推荐算法（反馈闭环）。
  - 周报中的寿命预警会引导用户回到生态系统或策略工坊，进行策略替换。

## 九、回测与实验：验证“这个策略/组合到底行不行”

**页面**：
- `/backtests` → [BacktestCenter.vue](frontend/src/views/BacktestCenter.vue)
- `/backtests/:backtest_id` → [BacktestDetail.vue](frontend/src/views/BacktestDetail.vue)
- `/ab-tests` → [ABTestDashboard.vue](frontend/src/views/ABTestDashboard.vue)

### 意义与作用
- **产品定位**：任何投资建议都必须经过历史验证。回测中心是用户验证策略和组合有效性的主战场。
- **给用户什么**：
  - **回测中心**：选择组合、时间范围、初始资金，批量运行多个策略的回测。
  - **回测详情**：策略 vs 买入持有基准的 11 项指标对比、K 线买卖点图、策略生态快照（健康/代谢/寿命/家族）、原始日志。
  - **A/B 测试**：以统计学方式比较两个策略，给出 p 值、胜率等指标，帮助用户做数据驱动的选择。
- **为后续功能提供什么关键输入**：
  - **今日操作 / 周报**：只有跑过回测且表现良好的策略，才会进入每日操作建议和周报绩效计算。
  - **策略匹配 / 生态系统**：回测结果是策略健康度和推荐排序的重要依据。
  - **模拟盘监控**：回测通过的策略，下一步可以放入模拟盘做真实行情验证。

## 十、模拟盘监控：在真实资金前先做“演习”

**页面**：`/paper-trading` → [PaperTradingMonitor.vue](frontend/src/views/PaperTradingMonitor.vue)

### 意义与作用
- **产品定位**：回测是历史验证，模拟盘是接近真实市场的验证。让用户在不亏钱的情况下，观察策略在实时行情中的真实表现。
- **给用户什么**：
  - 模拟交易会话列表及运行状态。
  - 模拟持仓、盈亏、交易信号的实时展示。
  - 策略流自动执行：选股 → 风控 → 交易的完整链路。
- **为后续功能提供什么关键输入**：
  - **真实持仓同步**：模拟盘的运行逻辑是未来接入真实券商成交数据的基础。
  - **策略生态系统**：模拟盘运行中的表现数据会补充策略健康度和寿命评估。
  - **周报 / 今日操作**：模拟盘结果可用于生成更贴近实际的交易建议。

## 十一、策略工坊：让用户创建和编辑自己的策略

**页面**：
- `/strategies` → [StrategyLayout.vue](frontend/src/views/strategies/StrategyLayout.vue)
- `/strategies/picker` → [PickerEditor.vue](frontend/src/views/strategies/PickerEditor.vue)
- `/strategies/trade` → [TradeEditor.vue](frontend/src/views/strategies/TradeEditor.vue)
- `/strategies/risk` → [RiskEditor.vue](frontend/src/views/strategies/RiskEditor.vue)
- `/strategies/flow` → [FlowEditor.vue](frontend/src/views/strategies/FlowEditor.vue)

### 意义与作用
- **产品定位**：为有一定能力的用户提供的策略创作工具。用户不仅可以使用平台推荐策略，还能自己写选股、交易、风控逻辑，并把它们编排成可执行的策略流。
- **给用户什么**：
  - **选股器**：编写 Python 选股策略，运行后得到股票池。
  - **交易引擎**：编写交易策略，支持可视化流程编辑和代码编辑两种模式。
  - **风控引擎**：编写风险检查逻辑，并用表单配置仓位上限、日最大回撤、黑名单等规则。
  - **策略流**：把选股、风控、交易策略编排为流水线，实现自动化执行。
- **为后续功能提供什么关键输入**：
  - **策略生态系统 / 基因报告**：用户创建的策略会被自动分析 DNA、家族、寿命等特征。
  - **回测中心 / 模拟盘**：策略创建后可以立即回测或上模拟盘。
  - **策略匹配 / 策略推荐**：优质用户策略有机会进入推荐池，被其他画像用户匹配使用。

┌─────────────────────────────────────────────────────────────────────────────┐
│                              投资者画像                                       │
│  （回答 15 道题 → 生成风险标签与投资风格）                                       │
└───────────────────────┬─────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────────┐

└───────────────────────┬─────────────────────────────────────────────────────┘
                        │
                        ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│                              投资组合生成                                       │
│  （画像 + 市场信号 → SAA/TAA 资产配置 + 具体标的绑定）                          │
└──────────┬───────────────────────────────────────────────┬──────────────────┘
           │                                               │
           ▼                                               ▼
┌─────────────────────┐                        ┌─────────────────────────────┐
│    策略匹配与执行    │                        │        建仓引导              │
│ （资产 → 策略 → 回测）│                        │  （开户 → 分阶段买入清单）      │
└──────────┬──────────┘                        └─────────────┬───────────────┘

           │                                                  │
           ▼                                                  ▼

每个页面都不是孤立存在，它们共同构成了“了解用户 → 判断市场 → 生成组合 → 匹配策略 → 回测验证 → 建仓执行 → 每日监控 → 长期迭代”的完整投资闭环。
