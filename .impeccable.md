## Design Context

### Users
Chinese quantitative traders and algorithmic strategy developers who use the app for extended trading sessions. Primary context is desktop/laptop screens in home offices or trading rooms. Users need to quickly write strategies, run backtests, and monitor paper trading signals.

### Brand Personality
- **Precise**: Every number, label, and interaction must feel exact and trustworthy
- **Technical**: Code-heavy interface that respects the user's engineering mindset
- **Focused**: Minimal distraction, information-dense where it matters

### Aesthetic Direction
- **Theme**: Dark mode (trading terminals are traditionally dark to reduce eye strain during long sessions)
- **Visual tone**: Refined industrial/utilitarian — clean lines, subtle depth, purposeful density
- **Colors**: Deep navy-charcoal backgrounds (#0d0f14, #141821) with warm amber accents (#f59e0b oklch tones) for a trading-floor feel. Avoid cyan-purple gradients.
- **Typography**: JetBrains Mono for code and data; Noto Sans SC for Chinese UI text.
- **Anti-references**: No glassmorphism, no neon glows, no gradient text, no card monoculture

### Design Principles
1. **Information density over whitespace** — traders scan data quickly; don't waste space
2. **Code is first-class** — the strategy editor should feel like a real IDE
3. **Status at a glance** — use color and position to communicate running/failed/success states instantly
4. **Dark by default** — reduce eye strain for extended use
5. **No decorative noise** — every visual element must serve a function
